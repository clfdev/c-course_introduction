Por que estudar Arduino depois de C?

Você chegou até aqui sabendo declarar structs, alocar memória com malloc, montar listas encadeadas e persistir dados em arquivos. Isso não é pouca coisa — a maioria dos cursos de Arduino começa do zero com "o que é uma variável". Você não precisa disso.

O que muda agora é o destino do código: em vez de rodar num computador e imprimir no terminal, seu programa vai rodar num microcontrolador e controlar o mundo físico — acender LEDs, mover motores, ler sensores de temperatura, detectar movimento.

A grande revelação: Arduino é C com alguns nomes diferentes. O chip do Arduino Uno (ATmega328P) executa código C compilado. A IDE do Arduino simplesmente esconde o processo de compilação e usa uma biblioteca de funções prontas para facilitar o acesso ao hardware. Tudo que você aprendeu — ponteiros, structs, funções, arrays — funciona dentro de um sketch Arduino.

O que é Arduino?

Arduino é uma plataforma de hardware + software open-source criada em 2005 na Itália para facilitar a prototipagem eletrônica. Ela tem duas partes que você precisa conhecer:

Parte O que é Analogia com o que você já conhece
Placa Arduino Um pequeno computador com microcontrolador, pinos de entrada/saída, porta USB e regulador de tensão Seu PC, porém sem tela, sem teclado e com pinos elétricos expostos
IDE Arduino Ambiente de programação onde você escreve, compila e envia o código para a placa O GCC + terminal que você usava para compilar e executar seus programas em C
Sketch Nome que o Arduino dá para um programa — é um arquivo .ino Seu arquivo .c
Biblioteca Conjunto de funções prontas para controlar um componente (servo, LCD, sensor) #include <stdio.h>, #include <string.h> etc.

Modelo mais popular: O Arduino Uno é o modelo de referência deste curso. Ele tem 14 pinos digitais, 6 pinos analógicos, 32 KB de memória flash para o programa, 2 KB de RAM e roda a 16 MHz. Para comparar: seu smartphone tem milhares de vezes mais memória e velocidade — mas para acender um LED ou ler um sensor, o Uno é mais do que suficiente.

A estrutura de um sketch: setup() e loop()

Todo sketch Arduino tem exatamente duas funções obrigatórias. Se você já trabalhou com main() em C, vai reconhecer o padrão imediatamente:

// Sketch Arduino mínimo — equivalente ao main() em C

void setup() {
    // Executada UMA VEZ quando a placa liga ou é resetada
    // Aqui você configura pinos, inicializa variáveis, abre comunicação serial
}

void loop() {
    // Executada REPETIDAMENTE para sempre, enquanto houver energia
    // Aqui fica a lógica principal do seu programa
}

Compare diretamente com o que você já conhece:

C puro Arduino O que acontece 
int main() {
    /* inicialização */

    while(1) {
        /* lógica */
    }
    return 0;
}
 
void setup() {
    /* inicialização */
}

void loop() {
    /* lógica */
}
 São equivalentes. O Arduino só divide o main() em duas funções nomeadas para deixar mais claro o que é inicialização e o que é o laço principal.

O que o Arduino faz por baixo dos panos: A IDE gera automaticamente um main() em C que chama setup() uma vez e depois fica chamando loop() dentro de um while(1). Você não vê esse código, mas ele existe. Quando compilamos um sketch, o compilador usado é literalmente o avr-gcc — um GCC para o chip AVR do Arduino.

C vs Arduino — mapa completo das diferenças

A tabela abaixo cobre tudo que você vai encontrar de diferente nas próximas aulas. Nada aqui é uma ruptura — é sempre uma adaptação do que você já sabe:

Conceito C puro Arduino Observação
Entrada/saída de texto printf() / scanf() Serial.print() / Serial.read() A saída vai para o Monitor Serial da IDE, não para um terminal
Pino digital (ligar/desligar) — (acesso direto a registradores) digitalWrite(pino, HIGH/LOW) HIGH = 5V, LOW = 0V
Leitura de pino digital — (acesso a registradores) digitalRead(pino) Retorna HIGH ou LOW
Leitura analógica — (requer ADC manual) analogRead(pino) Retorna 0–1023 (10 bits)
Espera/pausa sleep() / loops vazios delay(ms) / millis() delay(1000) = 1 segundo
Configurar pino — (registradores) pinMode(pino, OUTPUT/INPUT) Define se o pino envia ou recebe sinal
Tipo de arquivo programa.c programa.ino Internamente continua sendo C/C++
Tipos de dados int, float, char… Mesmos tipos + byte, word, boolean Os tipos C todos funcionam normalmente
Constantes booleanas true / false (C99) true / false + HIGH / LOW HIGH = 1, LOW = 0
Structs, ponteiros, arrays ✅ ✅ — exatamente iguais Tudo que você aprendeu funciona aqui
Funções ✅ ✅ — exatamente iguais Você pode criar quantas funções quiser além de setup() e loop()

Atenção ao tamanho dos tipos: No Arduino Uno (chip de 8 bits), int tem 2 bytes (não 4!), então vai de -32768 a 32767. Se precisar de números maiores, use long (4 bytes). Isso não acontecia no seu PC onde int é 4 bytes.

O primeiro sketch: Blink

O "Hello, World!" do Arduino é fazer um LED piscar. O pino 13 do Arduino Uno já tem um LED soldado na própria placa — você não precisa de nenhum componente extra para rodar este exemplo:

// Blink — o "Hello, World!" do Arduino
// Faz o LED do pino 13 piscar a cada 1 segundo

// setup() é chamada UMA VEZ ao ligar a placa
void setup() {
    pinMode(13, OUTPUT);  // configura pino 13 como saída
}

// loop() é chamada REPETIDAMENTE para sempre
void loop() {
    digitalWrite(13, HIGH);  // liga o LED (5V no pino 13)
    delay(1000);              // espera 1000 ms = 1 segundo
    digitalWrite(13, LOW);   // desliga o LED (0V no pino 13)
    delay(1000);              // espera mais 1 segundo
}

Agora veja o equivalente exato em C puro, para sentir o quanto são parecidos:

// Equivalente conceitual em C puro (pseudocódigo de hardware)
// Isso é O QUE O ARDUINO FAZ por baixo dos panos

#include <avr/io.h>
#include <util/delay.h>

int main() {
    DDRB |= (1 << PB5);   // configura pino 13 (PB5) como saída

    while (1) {
        PORTB |= (1 << PB5);   // liga o LED
        _delay_ms(1000);
        PORTB &= ~(1 << PB5);  // desliga o LED
        _delay_ms(1000);
    }
    return 0;
}

Conclusão visual: O Arduino esconde a manipulação de bits nos registradores (DDRB, PORTB) e oferece funções com nomes legíveis (pinMode, digitalWrite). A lógica é idêntica — o que muda é o vocabulário.

Boas práticas desde o início: constantes e nomes

Assim como em C você evitava números mágicos e usava #define, no Arduino a mesma prática se aplica. A plataforma já define a constante LED_BUILTIN para o LED da placa:

// Versão com boas práticas

#define INTERVALO 500  // ms — fácil de mudar

void setup() {
    pinMode(LED_BUILTIN, OUTPUT);  // LED_BUILTIN = 13 no Uno
    Serial.begin(9600);            // abre comunicação serial a 9600 bps
    Serial.println("Arduino iniciado!");
}

void loop() {
    digitalWrite(LED_BUILTIN, HIGH);
    Serial.println("LED ligado");
    delay(INTERVALO);

    digitalWrite(LED_BUILTIN, LOW);
    Serial.println("LED desligado");
    delay(INTERVALO);
}

Boa prática: Sempre use Serial.begin(9600) no setup() quando quiser depurar seu código. É o equivalente ao printf() que você usava para acompanhar o estado das variáveis — indispensável para entender o que está acontecendo no hardware.
