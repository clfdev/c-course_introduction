O que é o Wokwi?

Wokwi (wokwi.com) é um simulador de Arduino online e gratuito. Você escreve o código, monta o circuito e vê tudo funcionando — sem precisar de placa, LED, resistor ou fio físico nenhum.

Por que usar o Wokwi antes do hardware físico?

  • Você não queima componentes ao errar a fiação
  • Testa o código instantaneamente, sem compilar e enviar para a placa
  • Os projetos ficam salvos na nuvem — acessíveis de qualquer computador
  • Permite pausar a simulação e inspecionar o estado dos pinos em tempo real
  • Tem Monitor Serial integrado — igual ao da IDE do Arduino

A estratégia deste curso é: simule no Wokwi primeiro, entenda o comportamento, e só depois monte no hardware físico. Você vai errar menos, aprender mais e não vai desperdiçar componentes.

O circuito do Blink com LED externo

O LED soldado na placa (pino 13) é ótimo para testar, mas na prática você vai sempre usar LEDs externos conectados por fios. Isso exige entender dois componentes novos:

Componente O que é Por que é necessário
LED Diodo Emissor de Luz — acende quando corrente elétrica passa no sentido correto É o componente que vai acender. Tem polaridade: a perna longa é o ânodo (+) e a curta é o cátodo (−)
Resistor 220Ω Limita a corrente elétrica que passa pelo LED Sem ele, o LED recebe corrente demais e queima em segundos. O Arduino fornece 5V; o LED aguenta ~2V com ~20mA — o resistor absorve a diferença
GND (Terra) O pino de referência 0V da placa Todo circuito precisa de um caminho de retorno para a corrente. Sem conectar o GND, nada funciona

Regra do resistor: Sempre que conectar um LED a um pino do Arduino, use um resistor de 220Ω a 470Ω em série. Isso vale no Wokwi (simulação mais realista) e, especialmente, no hardware físico (proteção real).

A ligação segue esta ordem:

Pino 13 do Arduino
    │
    ├─── Resistor 220Ω ─── ânodo LED (+, perna longa) ─── cátodo LED (−, perna curta)
                                                                    │
                                                                   GND

Código completo: Blink com LED externo e Serial

Este é o primeiro sketch completo do curso. Cada linha está comentada com o raciocínio que você já usa em C:

/*
 * Projeto 01 — Blink com LED externo
 * Pino 8: LED + resistor 220Ω → GND
 * Monitor Serial: exibe o estado atual do LED
 */

#define PINO_LED  8     // define o pino — como #define MAX 50 em C
#define TEMPO_ON  1000  // tempo ligado em ms
#define TEMPO_OFF  500  // tempo desligado em ms (pisca assimétrico)

// Variável global que conta quantas vezes o LED acendeu
int contadorPiscadas = 0;

// setup() — executada UMA VEZ ao ligar/resetar
void setup() {
    pinMode(PINO_LED, OUTPUT);   // pino 8 vai enviar sinal
    Serial.begin(9600);          // abre porta serial a 9600 bps
    Serial.println("=== Blink iniciado ===");
}

// loop() — executada REPETIDAMENTE para sempre
void loop() {
    // Acende o LED
    digitalWrite(PINO_LED, HIGH);
    contadorPiscadas++;
    Serial.print("Piscada #");
    Serial.print(contadorPiscadas);
    Serial.println(" — LED ligado");
    delay(TEMPO_ON);

    // Apaga o LED
    digitalWrite(PINO_LED, LOW);
    Serial.println("             LED desligado");
    delay(TEMPO_OFF);
}

             LED desligado

...

Agora a dissecação linha a linha das funções novas:

Linha O que faz Equivalente em C puro
pinMode(8, OUTPUT) Configura o registrador de direção do pino 8 para saída DDRB |= (1 << PB0)
Serial.begin(9600) Inicializa a UART do chip a 9600 bits por segundo Configurar os registradores UBRR0H/L e UCSR0B manualmente
digitalWrite(8, HIGH) Coloca 5V no pino 8 — acende o LED PORTB |= (1 << PB0)
digitalWrite(8, LOW) Coloca 0V no pino 8 — apaga o LED PORTB &= ~(1 << PB0)
delay(1000) Bloqueia o programa por 1000 ms usando o timer do chip _delay_ms(1000) do avr-libc
Serial.print("texto") Envia string pela UART (USB → Monitor Serial) printf("texto")

O problema do delay() e a solução com millis()

delay() é simples e funciona bem para exemplos básicos, mas tem um problema sério: ele bloqueia tudo. Enquanto o programa espera no delay(1000), ele não faz mais nada — não lê sensores, não verifica botões, não responde a nada.

A solução é millis(), que retorna o número de milissegundos desde que a placa ligou — como um relógio. Com ele você verifica se já passou tempo suficiente sem parar o programa:

// Blink SEM delay() — usando millis()
// O programa continua livre para fazer outras coisas

#define PINO_LED    8
#define INTERVALO   1000  // ms entre mudanças de estado

int estadoLED = LOW;
unsigned long ultimoTempo = 0;  // guarda o momento da última mudança

void setup() {
    pinMode(PINO_LED, OUTPUT);
    Serial.begin(9600);
}

void loop() {
    unsigned long agora = millis();  // que horas são agora?

    // Passou tempo suficiente desde a última mudança?
    if (agora - ultimoTempo >= INTERVALO) {
        ultimoTempo = agora;  // atualiza o "relógio"

        // Inverte o estado do LED
        estadoLED = (estadoLED == LOW) ? HIGH : LOW;
        digitalWrite(PINO_LED, estadoLED);

        Serial.print("millis=");
        Serial.print(agora);
        Serial.println(estadoLED == HIGH ? "  LED ON" : "  LED OFF");
    }

    // Aqui você pode fazer OUTRAS COISAS enquanto espera
    // ler sensores, verificar botões, atualizar display…
}

Regra de ouro: delay() serve para exemplos simples e testes rápidos. Em projetos reais com mais de um componente, use sempre millis(). Você vai entender o motivo na prática quando chegar nos projetos com sensores e motores simultâneos.

Aspecto delay(ms) millis()
Bloqueia o programa? Sim — programa congela pelo tempo inteiro Não — programa continua rodando normalmente
Facilidade de uso Muito simples Requer variável de controle de tempo
Quando usar Exemplos simples, um único componente Sempre que o projeto tiver 2+ componentes simultâneos
Analogia em C sleep() Verificar time(NULL) num loop sem bloquear

Montando o projeto no Wokwi passo a passo

Use este roteiro para montar o circuito do Blink no Wokwi agora mesmo:

  1. Acesse wokwi.com → New Project → Arduino Uno
  2. Clique no "+" e adicione um LED (vermelho, verde ou azul — sua escolha)
  3. Clique no "+" e adicione um Resistor. Na propriedade value, coloque 220 (ohms)
  4. Conecte os fios:
    • Pino 8 do Arduino → terminal esquerdo do Resistor
    • Terminal direito do Resistor → ânodo do LED (perna longa, lado +)
    • Cátodo do LED (perna curta, lado −) → GND do Arduino
  5. Cole o código da seção anterior na aba sketch.ino
  6. Clique em ▶ Play e veja o LED piscar
  7. Abra o Monitor Serial e acompanhe as mensagens

Atalho útil no Wokwi: Clique no LED durante a simulação para ver sua tensão e corrente em tempo real. Isso ajuda a entender o que está acontecendo no circuito mesmo sem um multímetro.
